Lista Endpointów API

Serwer działa pod adresem: http://localhost:5000

Kliknij na linki, aby otworzyć endpointy typu GET w nowej karcie i zobaczyć ich odpowiedź (zazwyczaj w formacie JSON).

Uwaga: Endpointy typu POST nie mogą być wywołane przez proste kliknięcie linku (które zawsze wykonuje zapytanie GET). Są one wymienione dla kompletności, ale do ich przetestowania potrzebne jest narzędzie takie jak Postman, curl, lub interfejs frontendowy aplikacji, który wysyła odpowiednie dane.

I. Strony HTML (Frontend)

  • GET / - Serwuje główną stronę aplikacji. Otwórz
  • GET /alarms - Wyświetla stronę z historią wszystkich alarmów. Otwórz
  • GET /operacje - Wyświetla stronę z operacjami tankowania. Otwórz
  • GET /filtry - Renderuje stronę z panelem monitoringu filtrów. Otwórz
  • GET /cykle-monitor - Serwuje stronę monitoringu cykli filtracyjnych. Otwórz
  • GET /aktywne-partie - Serwuje stronę zarządzania aktywnymi partiami. Otwórz

II. API - Sprzęt, Topologia i Zasoby

  • GET /api/sprzet - Zwraca listę całego sprzętu WRAZ z informacją o znajdującej się w nim partii. Zobacz JSON
  • GET /api/zawory - Zwraca listę wszystkich zaworów i ich aktualny stan. Zobacz JSON
  • GET /api/punkty_startowe - Zwraca listę wszystkich portów wyjściowych (OUT). Zobacz JSON
  • GET /api/punkty_docelowe - Zwraca listę wszystkich portów wejściowych (IN). Zobacz JSON
  • GET /api/sprzet/filtry - Zwraca listę filtrów. Zobacz JSON
  • GET /api/topologia - Zwraca pełną listę połączeń (segmentów) do wizualizacji. Zobacz JSON
  • GET /api/typy-surowca - Zwraca listę dostępnych typów surowca. Zobacz JSON
  • GET /api/sprzet/beczki-brudne - Zwraca listę beczek brudnych dostępnych do tankowania. Zobacz JSON
  • GET /api/sprzet/reaktory-puste - Zwraca listę reaktorów w stanie 'Pusty'. Zobacz JSON
  • GET /api/sprzet/reaktory-z-surowcem - Zwraca listę reaktorów zawierających surowiec. Zobacz JSON
  • GET /api/sprzet/pomiary - Zwraca listę sprzętu z ostatnimi odczytami temperatury i ciśnienia. Zobacz JSON

III. API - Główne Operacje i Procesy

  • POST /api/operacje/tankowanie - Tworzy nową partię przez tankowanie. (Wymaga danych, testuj przez Postman)
  • POST /api/operacje/rozpocznij_trase - Rozpoczyna operację transferu surowca. (Wymaga danych, testuj przez Postman)
  • POST /api/operacje/zakoncz - Kończy aktywną operację transferu. (Wymaga danych, testuj przez Postman)
  • POST /api/zawory/zmien_stan - Zmienia stan pojedynczego zaworu. (Wymaga danych, testuj przez Postman)
  • POST /api/trasy/sugeruj - Sugeruje najkrótszą trasę, ignorując stan zaworów. (Wymaga danych, testuj przez Postman)
  • POST /api/operacje/dobielanie - Rejestruje operację dobielania partii. (Wymaga danych, testuj przez Postman)
  • POST /api/operacje/tankowanie-brudnego - Tankowanie brudnego surowca z beczki do reaktora. (Wymaga danych, testuj przez Postman)
  • POST /api/operacje/transfer-reaktorow - Transfer surowca z reaktora do reaktora. (Wymaga danych, testuj przez Postman)

IV. API - Monitoring i Status

  • GET /api/operacje/aktywne - Zwraca listę wszystkich aktywnych operacji. Zobacz JSON
  • GET /api/filtry/status - Zwraca aktualny status dla każdego filtra. Zobacz JSON
  • GET /api/filtry/szczegolowy-status - Pobiera szczegółowy status filtrów z informacjami o partiach i cyklach. Zobacz JSON
  • GET /api/alarmy/aktywne - Zwraca listę aktywnych alarmów. Zobacz JSON
  • GET /api/pomiary/historia - Pobiera historię pomiarów z ostatnich 24 godzin. Zobacz JSON

V. API - Zarządzanie Partiami i Cyklami

  • GET /api/partie/aktualny-stan - Pobiera aktualny stan wszystkich partii w systemie. Zobacz JSON
  • GET /api/partie/aktywne - Pobiera listę wszystkich aktywnych partii w systemie z pełnymi szczegółami. Zobacz JSON
  • GET /api/cykle-filtracyjne/<id_partii> - Pobiera historię cykli dla danej partii. (Wymaga podania ID partii w URL)
  • GET /api/partie/szczegoly/<partia_id> - Pobiera szczegółowe informacje o konkretnej partii. (Wymaga podania ID partii w URL)
  • POST /api/cykle/rozpocznij - Rozpoczyna nowy cykl filtracyjny dla partii. (Wymaga danych, testuj przez Postman)
  • POST /api/cykle/zakoncz - Kończy aktualny cykl filtracyjny. (Wymaga danych, testuj przez Postman)
  • POST /api/partie/aktualizuj-status - Aktualizuje status partii. (Wymaga danych, testuj przez Postman)

VI. API - Endpointy Testowe i Kontrolne

  • POST /api/alarmy/potwierdz - Potwierdza aktywny alarm. (Wymaga danych, testuj przez Postman)
  • POST /api/test/sensors - Wymusza odczyt czujników. (Nie wymaga danych, ale jest typu POST)
  • POST /api/test/alarm - Wymusza testowy alarm dla sprzętu. (Wymaga danych, testuj przez Postman)
  • POST /api/sprzet/<sprzet_id>/temperatura - Ustawia docelową temperaturę dla sprzętu. (Wymaga ID sprzętu i danych, testuj przez Postman)
